# DotDuel — GDD v2.2 (실행 가능 명세서)

**프로젝트**: 닷 아케이드의 4번째 게임. 시각장애 사용자를 위한 1v1 촉각·청각 카드 듀얼.
**개정**:
- v2.0 → v2.1: P0~P3 공백 모두 채움
- v2.1 → v2.2: 온라인 PvP를 v1.0에 포함 (Firebase 실시간 매칭 + 봇 폴백), Dicey Dungeons 스타일 UI 명세 추가
**플랫폼**: Web (Web Bluetooth/Serial) + Dot Pad
**작성일**: 2026-05-27

## 1. 코어 게임플레이 (P0 — 채움)

### 1.1 카드 풀 — 12종 스타터 셋

3개 카테고리(창 ▲ / 방패 ■ / 특수 ●). 닷패드에서 카테고리는 **하단 100셀의 대형 심볼**로 즉시 인지.

| ID | 이름 | 카테고리 | 수치 | 특수 효과 | 닷패드 심볼 |
|---|---|---|---|---|---|
| `s_wood` | 나무창 | 창 ▲ | 공 1 | — | 작은 ▲ (4셀) |
| `s_steel` | 강철창 | 창 ▲ | 공 2 | — | 중간 ▲ (8셀) |
| `s_long` | 장창 | 창 ▲ | 공 3 | 다음 턴 내 방어 -1 | 큰 ▲ (12셀) |
| `s_bolt` | 번개창 | 창 ▲ | 공 2 | 상대 방패 효과 무시 | 지그재그 ▲ |
| `s_twin` | 이중창 | 창 ▲ | 공 1+1 | 한 턴에 2회 타격 | 쌍 ▲ |
| `d_wood` | 목방패 | 방패 ■ | 방 1 | — | 작은 ■ |
| `d_iron` | 철방패 | 방패 ■ | 방 2 | — | 중간 ■ |
| `d_tower` | 타워방패 | 방패 ■ | 방 3 | 다음 턴까지 효과 지속 | 큰 ■ |
| `d_mirror` | 반사방패 | 방패 ■ | 방 1 + 반사 | 차감한 데미지의 절반(올림)을 상대에게 | 십자 ■ |
| `x_heal` | 응급치료 | 특수 ● | HP +2 | 단, 그 턴에 데미지 받음 | 작은 ● |
| `x_feint` | 속임수 | 특수 ● | — | 상대의 이번 턴 카드 공개를 다음 턴까지 지연 (정보 차단) | 점선 ● |
| `x_duel` | 결투 신청 | 특수 ● | 공 2 | 양쪽 모두에게 공 2 데미지 (자해 + 공격) | 큰 ● |

### 1.2 덱 / 손패 / 드로우

- **개인 덱 크기**: 16장 (위 12종 중 4종은 2장씩 중복, 디폴트 빌드는 자동 셔플)
- **시작 손패**: 3장
- **매 턴 시작 드로우**: 1장
- **손패 최대**: 5장 (초과 시 가장 오래된 카드 자동 폐기)
- **멀리건**: 게임 시작 시 1회 한해 손패 3장 통째 교체 가능 (F4 길게 누름)
- **덱 소진 시**: 매 턴 시작 시 "감속(Fatigue)" — 자동 1 데미지

### 1.3 HP / 승리 조건

- **시작 HP**: 10
- **최대 턴**: 15
- **승리**:
  1. 상대 HP 0 이하
  2. 15턴 종료 시 HP 높은 쪽 승리 (동률이면 카드 잔량 많은 쪽, 그것도 동률이면 무승부)
- **즉시 패배**: 자해/특수로 자신의 HP가 0 이하가 되면 즉시 패배 (이중 KO 가능 = 무승부)

### 1.4 전투 판정 표 — 양쪽 동시 공개

판정 순서: ①특수 효과 전처리 → ②공격 vs 방어 산출 → ③HP 적용 → ④사후 효과

| 내 카드 | 상대 카드 | 결과 |
|---|---|---|
| 창(X) | 창(Y) | 양쪽 모두 데미지: 나는 Y, 상대는 X |
| 창(X) | 방패(Y) | 상대 데미지 = max(X − Y, 0) |
| 창(X) | 응급치료 | 상대 데미지 X (치료는 그 턴 데미지를 못 막음, 단 +2 회복은 정산 후 적용) |
| 창(X) | 속임수 | 상대 데미지 X (속임수는 정보전용) |
| 창(X) | 결투신청 | 상대 데미지 X + 2 (결투의 추가 데미지 누적), 나는 2 데미지 |
| 방패 | 방패 | 무사 (양쪽 효과 무용) |
| 방패 | 응급치료 | 상대 +2 회복 |
| 방패 | 속임수 | 양쪽 무사 |
| 방패 | 결투신청 | 나는 max(2 − 방, 0), 상대는 2 |
| 응급치료 | 응급치료 | 양쪽 +2 회복 |
| 응급치료 | 속임수 | 나 +2 회복 |
| 속임수 | 속임수 | 양쪽 효과 발동 (다음 턴 양쪽 카드 공개 지연) |

**특수 키워드**:
- **번개창**: 방패 카드의 방어 수치를 무시. 즉, max(2−0,0) = 2 직격.
- **장창**: 다음 턴 내 방어 수치 −1 (반사방패는 0이 됨)
- **타워방패**: 두 턴 연속 방어 효과 (다음 턴 카드 선택 안 해도 방 3 적용, 단 다음 턴 공격 불가)
- **이중창**: 공 1을 두 번 (방패와 만나면 두 번 모두 정산. 방 2면 둘 다 막힘)
- **반사방패**: 차감한 데미지 ÷ 2 올림 → 상대에게 추가 데미지

### 1.5 매치 흐름

```
[로비] 
  → [매치 시작] → 멀리건 (선택) 
  → [턴 N 시작] → 드로우 1장 → 손패 announce 
  → [플래닝] (25초 디폴트 / 옵션 15·25·40) — 양쪽 동시 
  → [확정] 둘 다 F3 더블탭으로 commit 
  → [클래시 애니메이션] (1.5초 촉각 시퀀스) 
  → [정산 announce] (TTS로 결과 통보) 
  → [턴 N+1 시작] OR [게임 종료]
```

## 2. 닷패드 영역 분할 (P2 — 채움)

### 2.1 정수 행 분할

10행을 **3 / 4 / 3**으로 정수 분할 (총 30 × 10 = 300셀):

| 행 | 영역 | 용도 |
|---|---|---|
| 0-2 (3행) | **적 전선** (90셀) | 상대가 commit한 카드의 윤곽 + 카테고리 힌트 |
| 3-6 (4행) | **클래시 격전지** (120셀) | 공방 결과 애니메이션 (가장 넓게 — 핵심 정보) |
| 7-9 (3행) | **내 카드** (90셀) | 현재 포커싱 중인 내 카드의 큰 심볼 |

### 2.2 적 카드 정보량 — 카테고리 힌트로 결정 (P2 결정)

GDD v2.0의 "사각형 윤곽선만"은 정보 0이라 전략성 부재 → **카테고리 힌트** 도입:

| 상대 카드 카테고리 | 닷패드 상단 90셀 표기 |
|---|---|
| 비공개 (확정 전) | 점선 사각형 frame (dots 1+4 만 가로로) |
| 창 확정 ▲ | 사각형 frame + 가운데 **세로 막대** (창의 추상) |
| 방패 확정 ■ | 사각형 frame + 가운데 **가로 막대** (방패의 추상) |
| 특수 확정 ● | 사각형 frame + 가운데 **3×3 점 블롭** (모호한 mass) |

→ "상대가 창인지 방패인지 특수인지"는 알지만 **세부 카드는 모름**. 가위바위보 + 메타게임 성립.
→ **속임수** 카드 효과: 다음 턴 동안 이 힌트도 안 보이게 함 (정보 차단)

### 2.3 내 카드 (하단 90셀)

큰 심볼 + 수치를 핀 패턴으로:

```
. ▲ ▲ ▲ ▲ ▲ ▲ ▲ ▲ ▲ ▲ . . . . [공 2]
. ▲ ▲ ▲ ▲ ▲ ▲ ▲ ▲ ▲ ▲ . . . .
. ▲ ▲ ▲ ▲ ▲ ▲ ▲ ▲ ▲ ▲ . . . .
```

심볼 옆에 작은 점자(grade-1)로 수치 표기 가능하지만 점자 영역은 20셀에 별도 있으므로 그래픽 영역에는 굵은 심볼만.

### 2.4 클래시 애니메이션 (3-6행, 120셀, 1.5초)

3단계 시퀀스:

| Frame | 시간 | 패턴 |
|---|---|---|
| F1 | 0–500ms | 양 끝(좌/우)에 내·상대 심볼 등장 |
| F2 | 500–1000ms | 가운데로 모이며 충돌 (가운데 4×4 핀 빽빽이) |
| F3 | 1000–1500ms | 결과: 데미지 받은 쪽으로 핀 파동 흩어짐 |

내가 데미지를 받았으면 **하단으로 핀 파동**, 상대가 받았으면 **상단으로 파동**, 양쪽 모두 받으면 **양옆으로 파동**.

## 3. 20셀 점자 영역

### 3.1 모드 A (기본)

`MY:10 T:03 OP:08` (내 HP, 턴, 상대 HP) — 20셀 내에 영문 grade-1 점자로 압축.

### 3.2 모드 B (카드 탐색 중)

선택된 카드 이름 + 수치: `Steel Spear A2` (강철창 공2) — F2로 길게 누르면 효과 텍스트 스크롤 + 풀텍스트 TTS 낭독

### 3.3 모드 C (클래시 직후)

`-3 DMG to OP` 또는 `BLOCKED` 등 결과 압축 표기. 3초 후 모드 A로 자동 복귀.

## 4. 입력 매핑 (P3 — F2 분리 적용)

| 입력 | 기능 | 동작 |
|---|---|---|
| 좌 패닝키 | 이전 카드 포커스 | 손패에서 좌로. 첫 카드에서 더 누르면 비프 |
| 우 패닝키 | 다음 카드 포커스 | 손패에서 우로. 마지막에서 비프 |
| **F1 (탭)** | 전황 브리핑 | "현재 3턴, 내 HP 10, 상대 HP 8" TTS |
| **F1 (길게 1초)** | 손패 전체 announce | 손패에 있는 카드 3개를 순차 TTS |
| **F2 (탭)** | 점자 텍스트 스크롤 | 20셀 창에서 4칸씩 우로 스크롤 |
| **F2 (길게 1초)** | 카드 전체 TTS 낭독 | 풀텍스트 음성 (P3 분리) |
| **F3 (더블탭, 0.5초 내)** | 카드 확정 (Commit) | 금속체인 SFX + WAIT_OPPONENT 진입 |
| **F4 (탭)** | 선택 취소 / 모드 복귀 | 일반 탐색으로 회귀 |
| **F4 (길게 2초)** | 시스템 메뉴 | 볼륨/속도/점자모드/항복 |
| 가운데 팬키 (PanAll) | 마지막 안내 음성 반복 | TTS replay |

### 4.1 글로벌 단축

- **F1+F4 (동시 0.3초)** = 항복 확인창 ("정말 항복? F3로 확정")
- **F3+F4 (동시)** = 즉시 컨텍스트 헬프 (현재 페이즈에서 사용 가능한 키)

## 5. 게임 모드 (v1.0 — 멀티 + 봇 동시 출시)

### 5.1 모드 종류

| 모드 | 설명 | 매칭 |
|---|---|---|
| **빠른 듀얼** (디폴트) | 사용자가 "듀얼 시작" 누르면 자동으로 처리 | Firebase 큐에 push → 10초 대기 → 매칭 OR 봇 폴백 |
| **봇 듀얼** | 명시적으로 봇과만 | 즉시 시작 (쉬움/보통/어려움 선택) |
| **친구 듀얼** | 4자리 룸 코드 공유 | 둘 다 같은 코드 입력 시 매칭 |
| **튜토리얼** | 학습 모드 | 봇 (강제) |

### 5.2 매칭 흐름 (빠른 듀얼)

```
[유저 클릭 "듀얼 시작"]
   ↓
[Firebase /lobby/queue/ 에 본인 uid + timestamp 푸시]
   ↓
[10초 카운트다운 시작 — TTS "상대를 찾는 중... 5초"]
   ├── 매치 발견 (큐에 다른 유저 있음) → /matches/{id} 생성 → 양쪽 진입
   └── 10초 경과 → 큐에서 제거 → 봇 폴백 (보통 난이도)
                                      ↑
                  사용자에게 알림: "상대를 찾지 못했어요. AI와 듀얼합니다."
                  옵션: F4 = 큐로 돌아가기 (계속 매칭 시도), F3 = 봇 듀얼 시작
```

### 5.3 AI 봇 난이도

- **쉬움**: 무작위 카드 선택
- **보통** (기본): 가중 무작위 — 방패 50% / 창 35% / 특수 15%. HP < 4 일 때 방어 우선
- **어려움**: 위 + 플레이어의 최근 5턴 카드 사용 빈도를 학습해 카운터 가중치 적용

봇은 0.8~2.0초의 무작위 "생각하는 시간"을 가짐 (사람처럼 느끼게)

### 5.4 끊김 처리

- 상대 연결 30초 무응답 → 시스템 안내 "상대 연결이 끊어졌어요"
- 자동 봇 인계 옵션 제공 (HP·손패·턴 그대로 봇이 인계)
- 사용자가 거부 시 매치 무효 → 로비 복귀

### 5.5 닷패드 단절은 별도 처리

- 양쪽 모두 본인 닷패드를 본인 브라우저에 연결. 한쪽 닷패드만 끊겨도 상대는 영향 없음
- 닷패드 끊긴 쪽은 30초 그레이스 동안 게임이 정지 (PAUSE), 그 사이 점자 출력만 멈춤 — 화면+음성으로 계속 플레이 가능

## 6. 사운드 디자인 표 (P1 — 채움)

**원칙**: SFX = 카드 카테고리·액션의 시그너처. TTS = 명시적 정보. BGM = 분위기.

### 6.1 SFX 라이브러리

| 이벤트 | 사운드 묘사 | 주파수/길이 | 비고 |
|---|---|---|---|
| 카드 포커스 이동 | 짧은 클릭 | 800Hz, 80ms | 패닝 |
| 카드 확정 (F3 더블탭 성공) | 금속 체인 챙 | 200→2000Hz sweep, 300ms | commit 신호 |
| 창 공격 발동 | 휘파람 + 타격 | 휘파람 1.5kHz 200ms → 저음 80Hz 100ms | 공격 카테고리 시그너처 |
| 방패 막음 (블록 성공) | 둔탁한 챙 | 300Hz square 150ms | 방어 시그너처 |
| 특수 효과 발동 | 차임 (3음 분산화음) | C5-E5-G5 300ms | 마법성 |
| 데미지 받음 | 그르렁 + 저주파 | 100Hz sawtooth 400ms | 충격 |
| HP 1 남음 | 심장박동 루프 | 60Hz 단음 1초당 1회 | 긴장 BG |
| 턴 시작 종 | 종 1회 | 1.5kHz bell 400ms | 신호 |
| 타이머 5초 남음 | 시계 째깍 | 가속 비트, 점점 빠르게 | 긴박 |
| 승리 | 팡파레 (4음 상승) | C-E-G-C 600ms | 환희 |
| 패배 | 낮은 호른 | 100Hz 1.5초 | 무게감 |
| 무승부 | 중성음 (sine 2초) | 440Hz | 무미 |
| 오입력 비프 | 동글동글 (2음 하강) | 600→400Hz 100ms each | 경고 |
| 닷패드 단절 알림 | 고주파 반복 비프 | 2.5kHz 200ms / 800ms 간격 | 긴급 |

### 6.2 TTS 사용 규칙

- **자동 TTS 발화**: 턴 시작, 데미지/회복 알림, 게임 종료, 닷패드 키 가이드
- **수동 TTS 발화**: F1 (브리핑), F1 길게 (손패), F2 길게 (카드 풀텍스트)
- **음성 우선순위**: 시스템 알림 > 게임 결과 > 카드 정보 (낮은 우선순위는 큐잉, 이전 발화 끊지 않음)

### 6.3 BGM (옵션, 헤더에서 토글)

- 미니멀 드론 + 심박 루프 (60BPM 베이스)
- 클래시 직전 1초 톤 상승 (긴장)
- 클래시 직후 1초 톤 하강 (이완)

## 7. 튜토리얼 (P2 — 채움)

### 7.1 강제 튜토리얼 — 3단계 (최초 1회)

| Step | 시간 | 목표 | 내용 |
|---|---|---|---|
| **1. 영역 인지** | ~60초 | 닷패드 3영역 학습 | 음성: "위쪽 세 줄은 적, 가운데 네 줄은 격돌, 아래 세 줄은 너의 카드입니다. 닷패드 위로 손가락을 천천히 움직여 보세요." 각 영역에 정적 패턴 표시. F3로 진행. |
| **2. 카드 탐색** | ~90초 | 패닝키 + 점자 + F2 학습 | 손패 3장 (강철창/철방패/응급치료) 제공. 패닝키로 이동, 점자 읽기, F2로 풀텍스트 듣기 연습. F1로 브리핑 듣기. |
| **3. 1턴 모의 듀얼** | ~120초 | 확정 + 클래시 경험 | AI 봇과 1턴만. F3 더블탭으로 commit. 클래시 애니메이션 + 결과 듣기. |

### 7.2 스킵

- 1회 완료 후 메인 메뉴에 "튜토리얼 다시 보기" 옵션
- F1+F4 동시로 언제든 메인 메뉴 복귀

### 7.3 컨텍스트 헬프 (F3+F4)

현재 페이즈에 따라 동적 도움말 제공. 예:
- 플래닝 중: "패닝키로 카드 선택, F2 길게 풀텍스트, F3 더블탭으로 확정. 25초 안에 결정하세요."
- 클래시 중: "결과 발표 중. 곧 다음 턴이 시작됩니다."

## 13. 멀티플레이 아키텍처 (v1.0 포함)

### 13.1 기술 선택 — Firebase Realtime Database

**선택 이유**:
- 서버 코드 0 — 클라이언트만으로 매칭·동기화 가능
- 무료 티어로 동시 접속 100명, 일일 트래픽 10GB 충분 (시각장애 게이머 대상이라 트래픽 미미)
- 1초 미만 양방향 동기화 — 카드 듀얼에 적합한 레이턴시
- 일회성 익명 인증 — 가입 없이 바로 플레이

### 13.2 데이터 구조

```
firebase /
├── lobby /
│   └── queue / 
│       └── {pushId} : { uid, ts, mode: "quick" }     # 매칭 대기열
├── rooms /                                            # 친구 듀얼 룸 (4자리 코드)
│   └── {4-digit-code} :
│       ├── hostUid, guestUid, status, ts
└── matches /                                          # 진행 중 매치
    └── {matchId} :
        ├── createdAt, lastUpdate
        ├── turn: 3
        ├── phase: "PLANNING" | "WAIT" | "CLASH" | "OVER"
        ├── players /
        │   ├── A : { uid, hp, hand[], deck[], discard[], commit, ready }
        │   └── B : { uid, hp, hand[], deck[], discard[], commit, ready }
        ├── revealed / {turn} : { A: cardId, B: cardId }   # 공개된 카드 기록
        └── result : { winner, reason } | null
```

### 13.3 매칭 로직 (클라이언트 측)

```js
async function findQuickMatch() {
  const myEntry = { uid: myUid, ts: Date.now(), mode: 'quick' };
  // 1) 큐의 가장 오래된 다른 유저 찾기
  const queueRef = firebase.database().ref('lobby/queue');
  const snapshot = await queueRef.orderByChild('ts').limitToFirst(2).once('value');
  const entries = Object.entries(snapshot.val() || {});
  const opponent = entries.find(([id, e]) => e.uid !== myUid);
  
  if (opponent) {
    // 매치 발견 — 큐에서 양쪽 제거, /matches/{id} 생성
    const [oppId, oppData] = opponent;
    const matchId = await createMatch(myUid, oppData.uid);
    await queueRef.child(oppId).remove();
    return matchId;
  } else {
    // 큐에 본인 푸시 + 10초 타이머 시작
    const myPushId = await queueRef.push(myEntry);
    return new Promise((resolve) => {
      const timer = setTimeout(async () => {
        await queueRef.child(myPushId).remove();
        resolve({ matchId: null, fallback: 'bot' });
      }, 10_000);
      // 누군가가 나를 픽업하면 /matches 에 내 uid가 등장 — 감지
      const matchListener = firebase.database().ref('matches').orderByChild('players/B/uid').equalTo(myUid);
      matchListener.on('child_added', (snap) => {
        clearTimeout(timer);
        matchListener.off();
        resolve({ matchId: snap.key, fallback: null });
      });
    });
  }
}
```

### 13.4 상태 동기화

- 각 클라이언트는 `/matches/{id}` 전체를 listen
- 자기 차례 행동(`commit` 카드 선택)을 자기 `players/A` 또는 `B`에 write
- **클래시 정산은 양쪽 클라이언트가 동일 로직으로 계산** (덱·핸드는 양쪽 모두에게 동일하게 공개되니까 검증 가능)
- **신뢰 모델 (v1.0)**: 클라이언트 간 상호 신뢰. 치트는 v1.1에서 서버 측 검증 추가 (Firebase Cloud Functions 또는 Supabase)

### 13.5 친구 듀얼 (룸 코드)

```
1. 호스트가 "친구 듀얼" 누름 → 클라이언트가 4자리 랜덤 코드 생성 (예: "K7P3")
2. /rooms/K7P3 = { hostUid: meUid, ts, status: "WAITING" }
3. 호스트가 코드를 음성으로 친구에게 전달
4. 게스트가 "친구 듀얼" → 4자리 코드 입력 → /rooms/K7P3/guestUid 채움
5. 양쪽이 같은 매치 ID로 진입
6. 30분 미사용 시 자동 삭제 (Firebase 룰)
```

### 13.6 보안 / Anti-cheat (v1.0 최소)

- **익명 인증**: Firebase Anonymous Auth (UID 자동 발급)
- **레이트 리미트**: 1초당 5개 이상 commit 시 매치 강제 종료
- **카드 검증**: 양쪽 클라이언트가 카드 풀의 합법성 검증 (덱에 없는 카드 commit 시 매치 종료)
- v1.1: 서버 사이드 검증 (Firebase Cloud Functions)

### 13.7 닷패드 멀티플레이 시나리오

- **양쪽 다 닷패드 보유**: 표준 경우. 각자 본인 닷패드에 본인 손패·전황을 출력
- **한쪽만 닷패드 보유**: 닷패드 없는 쪽은 화면+음성으로 플레이. 양쪽 동등하게 진행됨
- **양쪽 다 닷패드 없음**: 화면+음성 듀얼 (시각장애 사용자 + 시각장애 사용자, 또는 시각장애 사용자 + 시각 사용자 보호자가 화면 보면서 도와주는 경우)
- 닷패드는 **각자 본인 브라우저에 본인의 디바이스를 연결** — 한 닷패드를 두 명이 공유하는 경우는 v1.0에서 미지원

### 14.3 핵심 컴포넌트

**A. 상단 스탯 행 (Stats Strip)**
- 한가운데 ATK / DEF / HP / TURN 4열
- 양옆에 두 플레이어 HP 하트 (가득 = 채워진 ♥, 빈 = 외곽선 ♡)
- 현재 턴 강조 표시

**B. 내 손패 패널 (좌측, 카드 list)**
- 카드 3-5장 세로 스택, 각 카드는 80×100px 픽셀아트 카드
- 현재 패닝키로 포커싱된 카드는 1.1배 확대 + 오렌지 글로우
- 카드 옆에 작은 영어 라벨 (SPEAR / SHIELD / SPECIAL)
- 풍선 텍스트로 효과 한 줄 표시

**C. 아레나 (중앙)**
- 두 듀얼리스트 캐릭터 (좌측 = 본인, 우측 = 상대)
- 클래시 직전: 양쪽이 가운데로 약간 다가옴 (애니메이션)
- 클래시 직후: 데미지 받은 쪽이 약간 뒤로 밀려나고 빨간 플래시
- 배경: 어두운 동굴 + 양옆 거대한 종유석/석순 (레퍼런스 이미지처럼)

**D. 상대 공개 카드 기록 (우측 패널)**
- 최근 3턴 동안 상대가 공개한 카드 = 작은 썸네일 stack (위 = 최신)
- 진행 중 턴(현재 commit한 카드는 아직 공개 안 됨)은 ? 표시

**E. 하단 키맵 strip**
- DotPuzzle / DotPet의 헤더 키맵과 동일 패턴
- F1 브리핑 / F2 카드설명 / F3 확정 / F4 취소 / 좌우 팬 = 이전·다음 카드

**F. 페이즈 인디케이터 (하단)**
- 현재 페이즈명 + 카운트다운 타이머 (남은 초)
- 타이머 < 5초이면 빨간 깜빡임 + SFX 가속

### 14.4 듀얼리스트 캐릭터 디자인

**좌측 — 여우 창잡이 (Fox Spearmaster)**
- 작고 날렵한 의인화 여우 캐릭터
- 가죽 갑옷 + 빨간 어깨망토
- 손에는 강철창
- 자세: 약간 앞으로 기울인 공격 자세
- 톤: Dicey Dungeons의 작은 캐릭터 (귀엽고 표정 있게)

**우측 — 곰 방패잡이 (Bear Shieldwarden)**
- 큰 의인화 곰 캐릭터
- 묵직한 철갑옷 + 골드 트림
- 손에는 큰 라운드 방패
- 자세: 안정적인 방어 자세, 약간 뒤로 비스듬
- 톤: 좌측보다 무게감 있음, 위협적이지만 우호적인 표정

> **참고**: 두 캐릭터의 비대칭(작고 날카로움 vs 크고 묵직함)은 창과 방패의 정서를 시각적으로 전달.

### 14.5 화면 vs 닷패드 출력 매핑

| 화면 | 닷패드 |
|---|---|
| 좌측 손패 list | 하단 90셀 — 현재 포커싱 카드 큰 심볼 |
| 우측 상대 기록 | 상단 90셀 — 가장 최근 상대 commit 카드 카테고리 힌트 |
| 중앙 아레나 클래시 | 중앙 120셀 — 충격 파동 애니메이션 |
| 상단 스탯 (HP/턴) | 20셀 점자 라인 — 압축 표기 |
| 하단 키맵 strip | (닷패드 자체 물리 키 — 시각 보조) |

→ 화면이 없어도 닷패드 + 음성으로 100% 플레이. 화면이 있으면 시각 보조자가 함께 즐길 수 있음.

### 14.6 애니메이션 / 모션

- **카드 포커스 이동**: 200ms 부드러운 vertical slide
- **카드 commit**: 카드가 아레나 중앙으로 날아감 (400ms)
- **클래시 임팩트**: 화면 전체 0.1초 빨간 플래시 + 화면 흔들림 (10px)
- **데미지 받음**: HP 하트가 빨갛게 깜빡이며 줄어듦
- **승리/패패**: 페이드아웃 → 결과 화면 페이드인

### 14.7 사운드와 동기화

화면 애니메이션과 SFX는 같은 타임스탬프에 발화. 시각장애 사용자에게는 SFX만으로 동일한 정보 전달.
